GHA → Harness Integration

The CI-to-CD handoff uses a Custom Webhook Trigger. After GHA completes build, scan, and signing, the final job sends a rich payload to Harness with full deployment context. 

# ci.yml → notify job
curl -X POST \
  -H 'content-type: application/json' \
  --url "${HARNESS_WEBHOOK_URL}" \
  -d '{
    "image_full": "index.docker.io/...:abc1234",
    "image_name": "tangerine-spring-boot-sample",
    "image_tag": "abc1234",
    "sha": "...", "branch": "main",
    "commit_message": "...",
    "run_url": "https://github.com/.../runs/..."
  }'

Harness maps payload fields to pipeline variables via an Input Set, making the pipeline fully dynamic and reusable across services.

Governance & Security

Harness Security Testing Orchestration (STO) normalizes results from any scanner into a unified dashboard. OPA policies act as automated gates throughout the pipeline.

OPA Policy Examples
  • • No Critical/High vulnerabilities
  • • Change window enforcement
  • • SLSA attestation verified
  • • Image signature required (Cosign)
  • • Approved registries only
  • • Naming & tagging conventions
  • • SBOM: allowed versions, packages, licenses
STO Scanners
  • • Secrets: Gitleaks
  • • Code Quality: SonarQube
  • • SAST: Semgrep
  • • SCA: OWASP, OSV Scanner
  • • Container: Snyk, Trivy
  • • CV: Dynatrace
  • • SBOM: CycloneDX

Scanner-agnostic — bring your own tools

What This Pipeline Demonstrates

Key capabilities showcased in this reference architecture

Supply Chain Security

CycloneDX SBOM generation, Cosign artifact signing, SLSA attestation, and component drift detection for every build.

Security Testing Orchestration

Unified vulnerability management across 6+ scanner types with normalized results and automated OPA policy gates.

Policy as Code (OPA)

Automated governance enforced at every pipeline stage, from vulnerability thresholds to change window compliance.

Progressive Delivery

Blue/Green and Canary deployment strategies with automated rollback on verification failure.

Continuous Verification

Release verification compares real-time metrics against baseline. Automatic rollback if deviation exceeds threshold.

ITSM Integration

ServiceNow Change Requests auto-created, approved, and closed. Jira tickets track the full deployment lifecycle.

Full Audit Trail

Every deployment tracked from commit to production. Trigger payload, scan results, and approvals logged in Jira.

Platform Flexibility

Supports both Harness CD pipelines and GitOps with ArgoCD. Deploy your way, the platform adapts to either model.

Agnostic Handoff

Webhook-based integration works with GitHub Actions, Artifactory, Google Artifact Registry, or any system that can fire a POST request.
